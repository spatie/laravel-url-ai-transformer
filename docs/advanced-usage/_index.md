This section covers advanced features and customization options.

- [Structured output](./structured-output): return validated, machine-readable data from a transformer
- [Customizing AI models](./customizing-ai-models): use different AI providers, models, and parameters per transformer
- [Customizing the stored result](./customizing-the-result): post-process the response or save extra data on the model
- [Conditional transformations](./conditional-transformations): control when transformers should run
- [Crawling URLs](./crawling-urls): automatically discover the URLs to transform
- [Exploring command options](./exploring-command-options): filter and control the transform-urls command
- [Regenerating results](./regenerating-results): re-run transformations for specific URLs
- [Handling failures](./handling-failures): inspect and recover from failed transformations
- [Listening for events](./events): hook into the transformation lifecycle
- [Using your own model](./using-your-own-model): extend the TransformationResult model
- [Customizing the job](./customizing-the-job): configure retries, backoff, and middleware
- [Overriding actions](./overriding-actions): customize how URLs are fetched and processed
- [Testing transformers](./testing-transformers): testing strategies for your custom transformers
